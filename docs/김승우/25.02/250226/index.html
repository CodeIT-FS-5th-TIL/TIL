트러블 슈팅​

RootLayout이 잘 있는데도 위와 같은 에러가 나는 문제

시도해 본 방법​

  • RootLayout에 <head> 태그 넣어보라고 함. - GPT → 실패
  • .next 파일 삭제 후, 다시 npm run dev 실행 → 실패

원인​

  • api 코드를 만들다가 app 폴더가 또 만들어졌었음..
  • src/app 폴더로 잡혀도 상위에 app 폴더가 있으면 Next.js가 app 폴더를 먼저 인식 함.

해결 방법​

  • app 폴더 삭제 후 api 폴더 이동!

배운 내용​

  • form, a 태그는 기본적으로 새로고침이 된다.
  • 여전히 mpa가 spa보다 휠씬 많다.
    • mpa가 개발 속도 빠르기는 하다. (PHP)
    • spa는 취업용!

철학이 있는 신입. (위시켓 프론트앤드 리드)

  • 이력서에 강점 어필.
    • 애매하게 말고
  • 나만의 끈질긴게 있어야 한다.
