## SQLD

### 1과목 모델링의 이해

- DB 모델링의 3단계

1. 개념적 데이터 모델링: 가장 높은 추상화 레벨, EA를 수립할 때 이용, 엔터티와 속성
2. 논리적 데이터 모델링: 데이터 모델에 대한 키, 속성, 관계등을 표현, 재사용성 높음, 정규화를 통해 데이터를 최소화, 식별자 도출 후 관계 정의
3. 물리적 데이터 모델링: 성능향상을 위한 반정규화, 테이블 인덱스 함수등을 생성

- 스키마 구조의 3단계

1. 외부 스키마: 사용자 관점, 다중 사용자 뷰 제공
2. 개념 스키마: 설계자 관점, 통합 뷰 제공
3. 내부 스키마: 개발자 관점, 물리적 뷰 제공

- ERD (IE 표기법)
  엔터티, 속성, 카디널리티(관계 참여도), 관계(실선: 식별 관계, 점선: 비식별 관계), 관계선택사양

1. 식별 관계: PK - FK
2. 비식별 관계: PK - PK
