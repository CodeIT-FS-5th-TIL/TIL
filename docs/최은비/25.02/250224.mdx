#라이브러리와 프레임워크의 차이
라이브러리는 우리가 불러와서 자유롭게 사용하는 것.코드를 자유롭게 쓸 수 있고 라이브러리를 사용하고싶을 때 사용할 수 있음.

- React 는 라이브러리.
  프레임워크는 우리의 코드를 불러오는 것. 특정한 규칙에 따라서 적절한 위치에 코드를 잘 작성하면 프레임워크가 코드를 불러와서 작동하게 한다.
- next js 프레임워크에선, react DOM 부분이 없다. 즉 커스톰할 수 없고, 우리가 할 수 있는건 pages 안에서 뭔가 만드는 것 뿐.
  -pages 폴더 안에 페이지(파일)를 만들면 컴포넌트이름과 상관없이 페이지 기반 라우팅이 됨.
  - 컴포넌트 export default 꼭 해줘야함.
    -> 라우팅 설정 따로 필요없는 편리함. 파일의 이름이 url로 사용됨으로 중요함.

어쩌면 당연한 기본 개념이었을지 모르겠지만 이걸 정확히 몰랐던 나에게는 오늘 'React는 라이브러리고 Next는 프레임워크'라는 점이 왜 Next를 사용하는게 불편하며 그럼에도 불구하고 사용해야하는지를 이해하는데 큰 도움이 되었다.
넥스트는 프레임워크이므로 정해진 프레임에 맞게 이미 지어진 집에 내가 더이상 집을 수리할 수 없고 적절한 위치에 코드들을 잘 작성해야하는 것이다.
그렇게 생각하고나니 이론과 개념들을 전부 이해하려고 너무 애쓰기보다 정해진 규칙대로 사용해보면서 익숙해지는게 가장 좋은 방법일 듯 하다.
